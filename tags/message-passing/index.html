Discomp_A06_Causality-and-Time

Notes

因果关系的确定

$\phi_1\xRightarrow{α}\phi_2$
意味着 $\phi_1$ 在因果关系上发生在 $\phi_2$ 之前，有三种定义：

  1. 两个计算事件发生在同一个处理器上，且先后发生
  2. $\phi_1$向$\phi_2$的处理器发送了m，且$\phi_2$是收到了m的计算事件
  3. 传递闭包

这显然构成了一个非自反偏序集

DisComp_A02_Message-Passing

Notes

基本形式化

本章介绍没有failure的message passing模型，同步和异步都会进行讨论。对于这类模型的复杂度，通常的衡量手段是时间复杂度和通信数量

specify一下一个概念:outbuf象征发出但还没有抵达的信息，inbuf象征已经抵达但还没有处理的消息，信息从outbuf发出，通过del事件传到inbuf，再通过comp事件进行处理。inbuf在初始时一定为空，outbuf可以不为空（因为规定了transition函数之后inbuf不能为空）

每一个节点在comp时，读取一个输入，根据转移函数跳转到下一个状态，输出为一个新的状态和每个outbuf（出信道）至多各一个输出信息

safety condition：任何执行序列的有限前缀中，归约的坏事不会发生。如果出了事，可以在确定的时间指出哪里发生了违反。比如死锁：“临界区不会有两个线程”

liveness condition：任何序列的无限部分将会可能包含期望的状态，关心最终的进展和终止。一件事情“一定会达成”，比如cpu starvation，可以定义为“如果cpu想进临界区，其在某个时刻一定会进入临界区”

如果一个配置序列满足safety，则其为一个execution。若一个execution满足liveness，则其为admissible execution

异步的定义：状态转移所需时间无上界

异步模型的admissible：每个节点有无穷的计算事件，同时每个发出的信息最终都会抵达。这不代表会陷入死循环，完成任务后陷入一个dummy loop也是termination的一种方式。形式化的，系统终止等价于所有节点进入一个终止状态（只会转移到自身）且不再有信息传递
